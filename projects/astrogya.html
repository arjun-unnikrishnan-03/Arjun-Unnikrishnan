01 — Problem

THE PROBLEM

Build a cross-platform Vedic + Western astrology SaaS from scratch: free + premium tiers, web + mobile clients sharing logic, paid AstrologyAPI-branded PDF reports, Razorpay payments with idempotent delivery, an authenticated AI chatbot with RAG on user-purchased PDFs, and zero tolerance for double-charging or missed report delivery.

02 — What I Built

IMPACT BULLETS

  • Architected and built a Turborepo monorepo spanning a Next.js 16 (App Router, React 19) web app and an Expo/React Native (SDK 54) mobile app sharing TypeScript packages for API client, Supabase auth, and design tokens — supporting both cookie-session (web) and Bearer-token (mobile) request authentication.
  • Engineered a 3-tier caching strategy (in-memory Map → Supabase Postgres → AstrologyAPI) reducing third-party calls to one per zodiac sign per day; integrated 60+ AstrologyAPI endpoints with Promise.allSettled parallel orchestration, AbortController timeouts, and retry-on-5xx with plan-restriction error discrimination.
  • Implemented a Razorpay payment pipeline with HMAC SHA-256 signature verification, webhook handlers for payment.captured/failed and refund.processed/failed, idempotent payment-to-report delivery using JSONB metadata lookups, and an admin-triggered refund flow with audit logging, email notifications, and automatic plan downgrade.
  • Designed a Supabase Postgres schema of 17 tables (profiles, payments, refunds, pdf_reports, birth_charts, audit_log, webhook_logs, etc.) with row-level security policies, GIN-indexed JSONB columns, auto-profile triggers on signup, and a custom increment_ai_credits RPC for atomic free-tier credit gating.
  • Built an AI chat + RAG system integrating AWS Cognito (with AES-256-GCM encrypted credential storage and mid-session token refresh + retry) and a custom RAG backend; implemented background fire-and-forget PDF indexing via parallel upload + summarize, persisting document_id for follow-up RAG queries on user-purchased reports.
  • Hardened the application for production with strict CSP/HSTS/X-Frame security headers, role-based middleware gating (admin/super_admin/banned), Resend-driven transactional email (12 templates including delivery-failure alerts), a 280+ URL programmatic sitemap, JSON-LD SEO, Vitest unit suites, and Playwright e2e coverage; deployed via Vercel and EAS Build.
03 — Result

THE RESULT

A SaaS platform that survives the messy reality of payments + AI + cross-platform clients: idempotent delivery so users are never double-charged or missing reports, token recovery so a single Cognito blip doesn't drop a paying user's chat session, and tight cost control via the 3-tier cache so I'm not bleeding money on third-party API calls.

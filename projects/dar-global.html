01 — Engagement

THE CONTEXT

This was a client engagement at WowLabz for DarGlobal — building the backend that powers a luxury real-estate booking platform: discovery across multiple property collections, secure multi-provider auth (phone OTP + Google + Facebook + Apple), KYC-driven multi-step bookings, portfolio management, real-time push, and tight Salesforce CRM integration — all with production-grade resilience, observability, and security.

Worked on backend services as part of the WowLabz engineering team. The bullets below describe my individual contributions and ownership areas within the broader project.

02 — My Contributions

IMPACT BULLETS

  • Built a TypeScript + Express 5 REST API (Node ≥18, Prisma 7, PostgreSQL, Redis) exposing 50+ endpoints across guest discovery, OTP/OAuth auth, user profile, booking workflow, portfolio, and admin RBAC — deployed with structured Pino logging and graceful SIGTERM/SIGINT shutdown.
  • Designed a multi-provider authentication system with Firebase Admin–verified Google/Facebook/Apple sign-in plus phone OTP, issuing rotating JWT access/refresh pairs (jose, HS256, JTI-tracked) with Redis-backed token blacklisting and Postgres-persisted refresh tokens for revocation.
  • Implemented Salesforce CRM integration using jsforce + a REST middleware client, including OAuth client-credentials auth, automatic re-authentication on INVALID_SESSION_ID with up to 4 retries, an Opossum circuit breaker (50% error threshold, 10s reset), and a Lead/Visit DTO ↔ domain adapter that hydrates user KYC, addresses, and tier on first OTP login.
  • Engineered a Redis caching layer with module-scoped, user-aware keys (sha256-hashed normalized URLs, sorted query params) and per-route TTLs (1 min – 7 days), exposing X-Cache HIT/MISS headers and cutting Strapi CMS hits for project/news/collaboration listings.
  • Modeled a Prisma/PostgreSQL schema for User, Admin, Role, RolePermission, AccessToken, OTP, UserAddress, UserSettings, RequestTracker — including bilingual (English/Arabic) name and address fields, KYC document fields, composite uniqueness on (countryCode, phoneNumber), and module/action permission matrices for admin RBAC.
  • Built a Strapi-backed multi-source content layer unifying projects from 6 distinct CMS endpoints (One-of-One, Amaya, Trump Plaza, Rayana, AIDA, Play Projects) with locale-aware routing, deep populate, and a shared circuit breaker for upstream resilience.
  • Implemented an async FCM push-notification pipeline with a durable RabbitMQ queue (amqplib, persistent messages) and a singleton Firebase Admin sender, decoupling notification fan-out from the request lifecycle.
  • Hardened the API surface with Helmet, CORS, Express rate limiting backed by Redis on auth-sensitive routes, Zod request validation, AsyncLocalStorage-based request-ID propagation, and a request-tracker middleware persisting method/URL/status/duration/UA/device metadata for audit.
  • Implemented timezone-correct visit scheduling (Asia/Dubai via Intl.DateTimeFormat) generating 30-minute slots with same-day past-slot filtering, and orchestrated scheduleVisit / rescheduleVisit / cancelVisit against Salesforce Leads with separate UTC vs. UAE time semantics for in-person vs. virtual visits.
03 — Outcome

THE OUTCOME

The areas I owned hold up under real-world conditions: resilient at integration boundaries (circuit breakers, retries, graceful re-auth), observable (structured Pino logs, request tracing via AsyncLocalStorage, audit trails), and secure by default (multi-provider auth, RBAC, rate limiting, request validation). The kind of code you can hand off to oncall.
